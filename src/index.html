A Guide To Animating Responsibly

Benefits of Using CSS Animations

One major benefit of using CSS animations is that you don't need to know JavaScript! Even if you do write JavaScript, you might not want to weigh your application down with unnecesasry extras when you need just a few minor enhancements. Most CSS animations often perform better than simple JavaScript animations. The browser itself can optimize and increase performance by adjusting the frame rate within its own rendering engine. This optimization allows most CSS animations to run more quickly and smoothly than their JavaScript counterparts.

Bounce Wit Me

So, what can you animate? Most elements on your page can be manipulated with animations and transformations. The sorcery behind these animations is in the keyframes that we assign to the elements. Keyframes are used to set the starting and ending states of each animation. The animation starts at 0% and ends at 100%, with any number of waypoints marked using any percentage in between.

Below you can see a few of the animation styles available next to the code that created them. Each example utilizes multiple subclasses in order to achieve the final effect.

Check me out! I'm bouncin' in while I get my colors on!

@keyframes bounce { 0% { transform: scale(0.1); opacity: 0; color: red; } 50% { color: green; } 100% { transform: scale(1); color: orange; } } .animate { animation-duration: 2s; animation-name: bounce; animation-iteration-count: infinite; animation-direction: alternate; color: orange; }
@keyframes spin { from { transform: rotate(0deg); } to { transform: rotate(360deg); } } .animateSpin { border-radius: 50%; animation: spin 3s infinite linear; }
